# 诚朴时间银行需求文档

## 一、项目背景与目标

### 1.1 项目背景

在社区生活、校园互助与公益服务场景中，大量真实需求与个人技能未被有效连接，主要原因包括：

- 金钱交易门槛高、信任成本高
- 公益与志愿行为缺乏持续激励机制
- 个体时间价值无法被量化与沉淀

“时间银行”作为一种互助模式，通过**以时间为计价单位**，将技能、服务与公益行为进行价值化和制度化管理，能够在一定程度上解决上述问题。

### 1.2 项目目标

诚朴时间银行旨在构建一个以“时间币”为核心的社区互助平台，实现：

- 将用户时间与技能转化为可流通的价值单位
- 通过任务机制连接需求方与服务方
- 以制度化的信用与仲裁体系保障协作安全
- 通过公益任务激励社区参与和社会价值创造

------

## 二、产品定位与用户画像

### 2.1 产品定位

- **产品类型**：社区互助型时间银行平台
- **核心模式**：任务发布 + 时间币结算
- **目标平台形态**：Web 应用（react 前端 + MySQL 后端）

### 2.2 目标用户

1. **普通用户**
   - 有技能、时间可提供服务
   - 或有生活 / 学习 / 技术等需求
2. **公益志愿者**
   - 参与官方公益项目
   - 通过志愿服务获得时间币激励
3. **平台运营人员（管理员）**
   - 负责内容审核、纠纷处理、系统运营

------

## 三、核心业务流程概览

1. 用户注册并完善个人资料、技能标签
2. 发布需求或在任务大厅接单
3. 双方确认任务 → 执行任务
4. 任务完成 → 时间币结算
5. 双向评价 → 更新信用分
6. 如产生纠纷 → 提交仲裁中心处理

------

## 四、功能模块详细需求

------

## 4.1 用户与身份管理模块

### 4.1.1 功能说明

提供用户身份体系，是所有业务功能的基础。

### 4.1.2 功能点

- 用户注册 / 登录（邮箱或手机号）
- 个人资料管理
  - 昵称、头像、简介
  - 技能标签（可多选）
- 用户等级 / 信誉积分展示
- 账号状态管理（正常 / 冻结）

### 4.1.3 数据要点（示例）

- User
  - id
  - nickname
  - skills[]
  - credit_score
  - created_at

------

## 4.2 需求发布与任务大厅模块

### 4.2.1 功能说明

作为平台核心入口，用于连接供需双方。

### 4.2.2 功能点

- 发布个人需求
  - 标题、描述
  - 所需技能
  - 预估时长（时间币）
- 任务大厅浏览
- 条件筛选
  - 技能类型
  - 时间币区间
  - 发布时间
- 查看任务详情

------

## 4.3 任务接单与流程管理模块

### 4.3.1 功能说明

规范任务从发布到完成的全流程。

### 4.3.2 任务状态流转

```
待接单 → 已接单 → 进行中 → 已完成 → 已评价
```

### 4.3.3 功能点

- 接单确认
- 任务进度更新
- 双方任务协作记录
- 任务取消 / 超时处理

------

## 4.4 时间币与账户资产模块

### 4.4.1 功能说明

构建平台内部的价值体系。

### 4.4.2 功能点

- 时间币余额展示
- 时间币获取
  - 完成任务
  - 公益奖励
- 时间币支出
- 时间币流水记录（账单）

### 4.4.3 设计原则

- 时间币仅用于平台内部流通
- 不支持直接兑换现金
- 防刷与异常检测由后台监控

------

## 4.5 公益项目与志愿活动模块

### 4.5.1 功能说明

由平台官方发布公益类任务，强化社会价值。

### 4.5.2 功能点

- 官方公益任务发布
- 志愿者报名 / 接单
- 完成后自动发放时间币奖励
- 公益项目展示专区

------

## 4.6 评价与信用机制模块

### 4.6.1 功能说明

通过评价体系建立信任基础。

### 4.6.2 功能点

- 双向评价（需求方 / 服务方）
- 星级评分 + 简要评价
- 信用分动态调整
- 信用等级展示

### 4.6.3 信用影响

- 影响任务推荐权重
- 低信用用户可能限制接单

------

## 4.7 消息通知与互动模块

### 4.7.1 功能说明

保障任务协作效率。

### 4.7.2 功能点

- 系统通知
  - 接单提醒
  - 状态变更
- 任务内消息聊天
- 仲裁结果通知

------

## 4.8 后台运营与数据管理模块

### 4.8.1 功能说明

用于平台管理与风险控制。

### 4.8.2 功能点

- 用户管理（封禁 / 解封）
- 任务内容审核
- 时间币流通监控
- 数据统计分析
  - 活跃用户数
  - 任务完成率

------

## 4.9 社区交流模块

### 4.9.1 模块目标

增强用户黏性，构建互助文化。

### 4.9.2 功能点

- 社区帖子发布
- 话题分类
  - 技能分享
  - 互助经验
  - 公益讨论
- 评论与互动
- 举报不良内容

------

## 4.10 争议仲裁中心模块

### 4.10.1 模块定位

保障任务交易公平性与安全性。

### 4.10.2 触发场景

- 任务未按约定完成
- 服务质量争议
- 时间币结算纠纷

### 4.10.3 功能点

- 提交仲裁申请
- 上传证据（文字 / 图片）
- 平台仲裁员处理
- 裁决结果执行
  - 时间币回滚
  - 信用分调整

------

## 五、技术架构说明（简要）

- **前端**：React
- **后端 / 数据库**：MySQL
